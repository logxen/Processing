Capture class
Name

read()
Description Captures a frame from the live video feed. The frame is returned as an array of bytes containing the image data for the frame. By default, the image data is in the .png file format and can be displayed by using loadImage() to create a PImage object.

Optionally, you can specify a width, height and alternative image encoding format for the capture. Not all phones support these options. Note that capturing high-resolution images often will cause out-of-memory problems in Java.
Syntax 
cap.read()
cap.read(width, height)
cap.read(encoding, width, height)
cap.read(paramstring)
Parameters
cap   Capture: any variable of type Capture
width   int: width of frame
height   int: height of frame
encoding   String: an alternative encoding, in mime format (i.e. image/jpeg)
paramstring   String: a custom parameter string as specified in the MIDP 2.0/MMAPI 1.0 specification
Returns byte[]
